PL-1112

Changes

Arrears only journey.

Hypothesis

We've seen:
The service doesn't apply TPD rules correctly (arrears only). Users are able to go through the entire service even if they will get a negative answer after the first question.

We believe this is because:
The service doesn't recognise that they have to be in 2 months of arrears to apply, and lets them carry on rather than knock them out at this stage.

So if we start:
Creating an end of journey for these users

We will see:
Less submissions which will get rejected straight away.

PL-954

Changes

Acknowledegment page content.

Hypothesis

We've seen Landlords don't know if their request has been received. We think this is because we don't give them enough information on the acknowledgement page. So if we update the acknowledgement page to give them more information. We'll see landlords feeling more comfortable once submitting the request in knowing it has been received.

Research notes

Tested with 10 users. The top three findings were:

  • Landlords understood that the request had ‘gone somewhere’ but further that that it was not clear what would happen next and when
  • The content did not make sense to all landlords
  • Neither Version drives the desired behaviours of: landlords feeling more assured / less calls or contact with DWP / no duplicate requests submitted

We will revert back to the original outcome page for further testing.
